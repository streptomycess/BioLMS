# Тренажер по биологии (Biology LMS)

Это веб-приложение на базе фреймворка Django, предназначенное для изучения биологии и проверки знаний. Платформа позволяет пользователям просматривать обучающие видео, закреплять материал с помощью карточек с вопросами (флеш-карточек) и отслеживать свой прогресс.

## Основные возможности

* **Видеоуроки:** Администраторы могут добавлять уроки со ссылками на видео из YouTube, которые автоматически встраиваются на страницу урока.
* **Интерактивные тесты:** К каждому уроку можно привязать набор вопросов с ответами (Flashcards) для проведения квизов.
* **Отслеживание прогресса:** Система сохраняет лучший результат пользователя (количество правильных ответов и общее количество вопросов) по каждому пройденному уроку.
* **Личный кабинет и статистика:** Предусмотрена система регистрации и авторизации пользователей, а также страница со статистикой успеваемости.

## Технологический стек

* **Backend:** Python, Django 6.0.3.
* **База данных:** SQLite3 (используется по умолчанию для локальной разработки).
* **Локализация:** Проект настроен на русский язык (`ru-ru`) и часовой пояс `Europe/Moscow`.

## Как запустить проект у себя локально

Следуйте этой пошаговой инструкции, чтобы развернуть и запустить проект на вашем компьютере. Убедитесь, что у вас установлен Python (рекомендуется версия 3.10 или выше).

### Шаг 1: Подготовка окружения

1. Откройте терминал (или командную строку) и перейдите в папку, где находится исходный код проекта (там, где расположен файл `manage.py`).
2. Создайте виртуальное окружение для изоляции зависимостей проекта:
   ```bash
   python -m venv venv
   ```
3. Активируйте виртуальное окружение:
   * **Windows:**
     ```bash
     venv\Scripts\activate
     ```
   * **macOS / Linux:**
     ```bash
     source venv/bin/activate
     ```

### Шаг 2: Установка зависимостей

Так как проект использует Django 6.0.3, установите его в ваше виртуальное окружение:
```bash
pip install django==6.0.3
```
*(Если в корне проекта есть файл `requirements.txt`, лучше выполнить команду `pip install -r requirements.txt`)*.

### Шаг 3: Настройка базы данных

В проекте используется легковесная база данных SQLite3. Чтобы создать необходимые таблицы для уроков, тестов и пользователей, выполните команду миграций:
```bash
python manage.py migrate
```

### Шаг 4: Создание администратора (суперпользователя)

Чтобы добавлять уроки и тесты через удобную панель управления, вам понадобится аккаунт администратора. Создайте его с помощью команды:
```bash
python manage.py createsuperuser
```
Система попросит вас ввести имя пользователя, email (можно пропустить, нажав Enter) и пароль. Пароль при вводе не отображается в целях безопасности.

### Шаг 5: Запуск сервера разработки

Теперь все готово для запуска приложения. Введите команду:
```bash
python manage.py runserver
```

### Шаг 6: Использование приложения

После успешного запуска сервера, приложение будет доступно в вашем браузере:

* **Главная страница приложения:** Откройте ссылку `http://127.0.0.1:8000/`. Здесь вы можете зарегистрировать обычного пользователя.
* **Панель администратора:** Откройте ссылку `http://127.0.0.1:8000/admin/` (если маршрут админки стандартный) и войдите с данными суперпользователя, которые вы создали на Шаге 4. В панели администратора вы можете создавать объекты `Lesson` (уроки) и добавлять к ним вопросы `Flashcard`.
